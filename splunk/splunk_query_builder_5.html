Lesson 01 · Fundamentals

What is SPL?

SPL (Search Processing Language) is Splunk's query language. Every SPL search is a pipeline: events flow left-to-right, each | passes the running result set into the next command.

copyindex=network sourcetype=cisco:ios host=leaf-01 | stats count by severity

Read it as: "pull events from the network index where sourcetype is cisco:ios and host is leaf-01, then count them grouped by severity."

Every SPL query has two halves:

  1. Retrieval — what events to pull from the index (before the first |).
  2. Transformation — what to do with them (everything after).
Rule of thumb: filter as much as you can in the retrieval half. It's dramatically faster than filtering after retrieval.
Lesson 02 · Basics

Search terms & quoting

Any bare word at the start of a query is a keyword match against raw event text.

copyindex=firewall denied

Multiple words = implicit AND:

copyindex=firewall denied tcp

Use quotes for phrases or values with spaces/special characters:

copyindex=syslog "interface Ethernet1/1 is down"
Gotcha: SPL is case-insensitive for keywords but case-sensitive for field values. error and ERROR both match as keywords; status=Error does not equal status=error.
Lesson 03 · Basics

Boolean logic & wildcards

SPL supports AND, OR, NOT (must be uppercase). Use parentheses to group.

copyindex=auth (failed OR denied) NOT user=admin

Wildcard * matches anything. Leading wildcards are expensive — avoid them.

copyindex=web status=5* user_agent=*bot*
Performance: *error is slow (leading wildcard). error* is fine. Prefer anchored patterns.

Lesson 05 · Basics

Time ranges

Use relative time modifiers with earliest / latest, or the time picker in the GUI. Syntax: -[n][unit][@snap].

UnitsMeaning
s, m, h, d, w, mon, yseconds, minutes, hours, days, weeks, months, years
@snap-to: truncates to unit boundary
nowright now
copyindex=syslog earliest=-24h@h latest=now severity=error

-24h@h = "24 hours ago, snapped to the start of that hour" — makes cached results reusable.

Tip: Snapping (@h, @d) makes Splunk cache better. Queries with identical snapped windows hit cache.
Lesson 06 · Pipelines

The pipeline (|)

Everything after the initial retrieval is chained with pipes. Each command takes the previous result set as input.

copyindex=web | stats count by status | sort -count | head 5

Pipeline: retrieve web events → aggregate count per status code → sort descending → take top 5.

Lesson 07 · Pipelines

fields, table, rename

fields keeps (or drops) columns early — good for performance. table is similar but enforces display order. rename relabels columns.

copyindex=network | fields _time host src_ip dest_ip action | rename src_ip AS "Source", dest_ip AS "Destination" | table _time host Source Destination action
Perf: fields early in the pipeline reduces data moved between search peers. Use it before heavy transformations.
Lesson 08 · Aggregation

stats & aggregations

stats is the workhorse transforming command. Syntax: stats [fn(field) AS alias]... [by field1, field2].

Common functions:

FunctionWhat it does
countnumber of events
count(field)events where field is non-null
dc(field) / distinct_countunique values of field
sum, avg, min, maxnumeric math
median, stdev, perc95statistical
values(field)multi-value list of unique values
list(field)all values, including duplicates, in order
first, last, earliest, latestvalue positions
copyindex=firewall action=blocked | stats count AS hits, dc(src_ip) AS unique_sources, sum(bytes) AS total_bytes, values(dest_port) AS ports_hit by host
Lesson 09 · Aggregation

top, rare, chart, timechart

top is shorthand for the most common pattern: count + sort + head.

copyindex=web | top limit=10 useother=f status

rare is the inverse — least common. chart is like stats but pivots into a two-dimensional table (good for visualizations).

copyindex=network | chart count over host by action

timechart buckets into time intervals — perfect for trend lines.

copyindex=firewall action=blocked | timechart span=5m count by dest_port limit=5
Lesson 10 · Transforming

eval & calculated fields

eval creates new fields or modifies existing ones using expressions, functions, and conditionals.

copyindex=web | eval response_kb = round(bytes/1024, 2) | eval is_error = if(status >= 400, "yes", "no") | eval tier = case( status < 300, "ok", status < 400, "redirect", status < 500, "client_error", true(), "server_error" )

Useful eval functions:

  • if(cond, yes, no) — ternary
  • case(c1,v1, c2,v2, ...) — multi-branch; use true() as the default
  • coalesce(a, b, c) — first non-null
  • len(s), lower(s), upper(s), replace(s, re, new)
  • strftime(_time, "%F %T") — format epoch to string
  • mvcount(mv), mvindex(mv, n) — multi-value handling
  • cidrmatch("10.0.0.0/8", ip) — CIDR tests (very handy for networking)
Lesson 11 · Filtering

where vs search

Both filter events. The difference matters:

  • search — can go at the start (implicit) or after a pipe. Matches fields and raw text. Runs on indexed data efficiently.
  • where — only after a pipe. Evaluates an expression; works with calculated fields from eval/stats. Supports eval functions like like(), isnotnull(), cidrmatch().
copyindex=firewall | stats count AS hits by src_ip | where hits > 100 AND cidrmatch("10.0.0.0/8", src_ip)
Rule: filter raw events with search; filter computed values with where.
Lesson 12 · Advanced

rex — regex extraction

rex pulls new fields out of existing text using named capture groups.

copyindex=syslog sourcetype=cisco:ios | rex field=_raw "%LINEPROTO-5-UPDOWN: Line protocol on Interface (?<intf>\S+), changed state to (?<state>\w+)" | stats count by host, intf, state

Syntax: (?<fieldname>pattern). Default field is _raw. Use max_match=0 to capture all matches into a multi-value field.

Replacement mode (mode=sed):

copy| rex field=url mode=sed "s/\?.*//"

…strips query strings from URLs.

Lesson 13 · Shaping

dedup, sort, head, tail

  • dedup N field1, field2 — keep at most N events per unique combo (default N=1)
  • sort [N] [+/-]field — + asc (default), - desc; optional limit
  • head N / tail N — first / last N events
  • reverse — reverse result order
copyindex=auth action=failure | dedup 3 user | sort - _time | head 50
Lesson 14 · Advanced

transaction

Groups related events into single "transactions" based on shared fields, time gaps, or start/end markers. Powerful but slow — use only when stats won't do.

copyindex=web sessionid=* | transaction sessionid maxspan=30m maxpause=5m | where duration > 600
When to avoid: if you can answer your question with stats ... by sessionid, use that. transaction keeps raw events in memory and scales poorly.
Lesson 15 · Advanced

Subsearches

A subsearch runs first and returns results used by the outer search, using [] brackets. Common pattern: "find all X where Y matches a list from another search."

copyindex=web status=500 [ search index=threat category=malicious | fields src_ip ]

The inner query returns IPs; outer treats that list as an OR filter on src_ip.

Limits: subsearches cap at 10,000 results and 60-second runtime by default. They're for lookups, not bulk joins.
Lesson 16 · Advanced

join & append

join SQL-style joins another search on a shared field. Slow — prefer stats when possible.

copyindex=web | stats count AS reqs by src_ip | join type=left src_ip [ search index=geo | stats values(country) AS country by src_ip ]

The stats equivalent (much faster when both searches share a field):

copy(index=web) OR (index=geo) | stats count AS reqs, values(country) AS country by src_ip

append runs a second search and concatenates its results at the end. Rarely the right tool; usually stats does the job.

Lesson 17 · Advanced

lookups

lookup enriches events with data from a CSV or KV store — great for mapping IPs to device names, MAC prefixes to vendors, etc.

copyindex=network | lookup devices.csv ip AS src_ip OUTPUT hostname AS src_host, role AS src_role | stats count by src_host, src_role, action

inputlookup reads a CSV as its own event source — useful for scheduled reports.

Lesson 18 · Patterns

Network engineering patterns

Interface flap detection

copyindex=syslog sourcetype=cisco:ios ("LINEPROTO-5-UPDOWN" OR "LINK-3-UPDOWN") | rex "Interface (?<intf>\S+?)," | bucket _time span=5m | stats count AS flaps by _time, host, intf | where flaps >= 4 | sort - _time

BGP neighbor state changes

copyindex=syslog ("BGP-5-ADJCHANGE" OR "BGP_NEIGHBOR_STATE_CHANGED") | rex "neighbor (?<peer>\d+\.\d+\.\d+\.\d+)" | rex "(?<state>Up|Down|Established|Idle|Active)" | stats count by host, peer, state | where state IN ("Down", "Idle", "Active")

Firewall top talkers (denied)

copyindex=firewall action=denied earliest=-1h | stats sum(bytes) AS total_bytes, dc(dest_ip) AS targets, count AS attempts by src_ip | where NOT cidrmatch("10.0.0.0/8", src_ip) | sort - total_bytes | head 20

VXLAN/EVPN — MAC moves

copyindex=syslog ("MAC_MOVE" OR "L2FM-4-L2FM_MAC_MOVE" OR "MAC-address moved") | rex "(?<mac>([0-9a-f]{2}[:.-]){5}[0-9a-f]{2}|([0-9a-f]{4}\.){2}[0-9a-f]{4})" | stats count AS moves, values(host) AS seen_on, dc(host) AS vtep_count by mac | where moves > 5 | sort - moves

Config changes (login + config events)

copyindex=syslog ("SYS-5-CONFIG_I" OR "CONFIG_I" OR "%PARSER-5-HIDDEN") | rex "by (?<user>\S+) on (?<tty>\S+) from (?<src>\S+)" | table _time host user src tty _raw | sort - _time
Lesson 19 · Operations

Performance tips

  1. Always specify index and time range. "All indexes, all time" is the cardinal sin.
  2. Push filters left. Filter in the initial search, not with where after.
  3. Use fields early to drop columns you don't need.
  4. Avoid leading wildcards (*foo). Trailing wildcards are fine.
  5. Prefer stats over transaction unless you truly need raw event grouping.
  6. Prefer stats over join when the two searches share a field — it's often 10×+ faster.
  7. Snap time ranges (-24h@h) to hit the cache.
  8. Use accelerated data models with tstats when doing volume metrics over long windows — orders of magnitude faster than stats.
copy| tstats count where index=firewall earliest=-7d@h by _time, action span=1h | where action="blocked"
tstats runs against indexed fields only, without reading raw events — dramatically faster for high-volume aggregations.
